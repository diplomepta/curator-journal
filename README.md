# Электронный журнал куратора — улучшенная версия

Проект сделан на Node.js, Express.js, EJS, Bootstrap 5 и MySQL.

## Что изменено

- Полностью переработан общий дизайн: новая навигация, карточки, главная панель, форма входа и единый стиль всех страниц.
- На главную добавлены быстрые действия, ближайшие мероприятия, группа риска и журнал последних действий.
- В разделе студентов добавлены фильтры по группе и статусу.
- В отчётах добавлен экспорт CSV по студентам и посещаемости.
- В мероприятиях добавлены обязательные поля: место проведения, ответственный, целевая аудитория и подробное описание.
- В карточке мероприятия добавлено удаление участников.
- Сделано больше обязательных полей в формах: студенты, представители, оценки, дисциплины, пользователи, мероприятия.

## Первый запуск с чистой базой

1. Установить Node.js и MySQL.
2. В MySQL/phpMyAdmin выполнить файлы:
   - `db/schema.sql`
   - `db/seed.sql`
3. Скопировать `.env.example` в `.env`.
4. В `.env` указать пароль от MySQL.
5. Установить зависимости:

```bash
npm install
```

6. Запустить сайт:

```bash
npm start
```

7. Открыть в браузере:

```text
http://localhost:3000
```

## Если база уже была создана раньше

Если ты уже импортировал старую базу и не хочешь её удалять, выполни файл:

```text
db/migration_improved.sql
```

После этого можно запускать сайт обычной командой:

```bash
npm start
```

Если данные не нужны, проще заново выполнить `schema.sql`, а потом `seed.sql`.

## Настройка пароля MySQL

В файле `.env` нужно указать пароль от MySQL:

```env
DB_PASSWORD=1302
```

Если MySQL работает без пароля, оставить строку пустой:

```env
DB_PASSWORD=
```
